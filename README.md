## 🧱 Clean Architecture (How I think about code)
- **Domain / Entities**: core business rules, independent of frameworks
- **Use Cases**: application-specific flows (create task, checkout order, etc.)
- **Adapters**: controllers, persistence, external services
- **Frameworks**: Spring Boot stays at the edge

**Goal:** keep the core logic easy to test and change — without fighting the framework.
